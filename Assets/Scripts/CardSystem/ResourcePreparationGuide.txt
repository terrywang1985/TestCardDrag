# 卡牌游戏资源准备指南

您看到的蓝色底面空白场景是因为**缺少卡牌图片资源**。本指南将帮助您快速准备和添加所需的图片资源。

## 为什么是空白的？

当前系统已经成功创建了所有游戏对象和逻辑，但由于没有提供卡牌的精灵图（Sprite）资源，所以卡牌无法正常显示。

## 需要准备的图片资源

您需要准备以下图片资源：

1. **卡牌背景图**：用于卡牌的基本外观
2. **卡牌图标**：各类卡牌的代表性图标
3. **测试图片**：用于快速测试游戏功能

## 图片规格建议

- **尺寸**：建议卡牌图片尺寸为 `300x400` 像素（宽x高）
- **格式**：支持 PNG、JPG 等常见图片格式，推荐使用 PNG（支持透明背景）
- **分辨率**：72-300 DPI 均可，根据实际需求调整

## 资源存放位置

请将所有图片资源放置在以下目录中：

```
Assets/Resources/CardSprites/
```

如果该目录不存在，请手动创建。

## 快速测试方法

如果您不想立即准备所有资源，可以使用以下**快速测试方法**：

1. 创建一个简单的测试图片（可以用任何图片，如纯白色方块）
2. 将图片命名为 `TestCard.png`
3. 放入 `Assets/Resources/CardSprites/` 目录
4. 重新运行游戏，系统会自动使用该图片作为测试卡牌的显示

## 让系统使用您的图片

系统默认会在 `CardSprites` 文件夹中查找卡牌图片。如果您想自定义图片加载逻辑，可以修改 `CardVisual.cs` 文件中的 `SetCardData` 方法。

## 创建测试卡牌数据

请确保您已经通过 `Card Game Tools > Create Test Card Data` 菜单选项创建了测试卡牌数据。

## 常见问题排查

1. 如果运行后仍然看不到卡牌：
   - 确认图片已正确放入 `Assets/Resources/CardSprites/` 目录
   - 确认图片名称正确且没有拼写错误
   - 确认图片格式受Unity支持（PNG、JPG等）

2. 如果卡牌显示异常：
   - 检查图片尺寸是否合适
   - 确认图片导入设置正确（在Unity中选择图片，检查Inspector面板）

## 资源优化建议

- 使用压缩格式减少内存占用
- 考虑为不同分辨率设备准备多套图片
- 对于大型游戏，可以考虑使用AssetBundle管理资源

如果您需要更详细的资源管理帮助或想进一步自定义卡牌外观，请参考Unity官方文档或联系技术支持。
